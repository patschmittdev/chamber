# Product

## Register

product

## Users

Operators use Chamber in a desktop workspace to create, coordinate, and extend
their AI minds. They need focused authoring tools that make it clear what will be
saved, where it applies, and when an active mind is drafting rather than changing
an artifact directly.

## Product Purpose

Chamber is an A2A-first desktop workspace for creating, running, and coordinating
AI agents powered by GitHub Copilot. Its authoring surfaces let operators safely
shape reusable prompts and mind-scoped skills without exposing implementation
details or bypassing the product's security boundaries.

## Brand Personality

Calm, capable, and precise. The interface should feel like a trustworthy desktop
tool that keeps operators in control of consequential agent configuration.

## Anti-references

Avoid tutorial-heavy screens, decorative dashboard styling, opaque automation,
and modal-first editing flows that obscure scope or save state.

## Design Principles

- Keep authoring focused on one artifact and its immediate context.
- Make scope, provenance, and mutable state explicit without exposing sensitive
  implementation details.
- Distinguish drafting with a mind from directly saving user-authored content.
- Prefer familiar, accessible desktop controls over novelty.
- Reveal detail progressively while preserving operator control.

## Accessibility & Inclusion

Use the existing accessible component primitives, clear keyboard navigation,
visible focus, text-backed status, and reduced cognitive load. Do not rely on
color alone for errors, state, or provenance.
